为 fortran-lang.org 做贡献#

Fortran-lang.org 是开源的，欢迎贡献！

  • 有关如何将条目添加到 包索引 的信息，请参见 packages

  • 请参阅 fortran-lang.org 的迷你书教程 了解如何为 Learn 部分编写和构建迷你书教程

介绍#

网站是怎么写的？

网站的内容主要是用 ReStructuredText、Markdown、HTML 和 YAML（用于数据）的组合编写的。此源代码经过编译以生成纯 HTML，这就是您在最终网站上看到的内容。

网站是_静态_的，这意味着一旦建立，网站上的内容对所有用户都是一样的；这与许多_动态_的网站形成对比，这意味着它们可以根据用户和用户提供的输入提供不同的内容。

网站的结构组件是用 Sphinx 静态网站生成器编写的，用于静态功能，而 JavaScript 用于动态功能。

我是否需要了解 HTML 才能做出贡献？

网站的大部分内容都是用 Markdown 编写的，这是一种用于格式化文本的简单标记语言 —— 如果您没用过，很容易上手！

网站是如何建成的？

Fortran-lang 站点使用基于 Python 的 Sphinx 静态站点生成器 来编译 RST、Markdown 和 HTML 文件。建议贡献者在您的开发计算机上安装 Python，以便可以在本地预览更改，但这不是强制性的，因为可以在拉取请求过程中生成站点预览（有关更多信息，请参见下文）。有关如何设置 Sphinx 和构建站点的信息，请参阅 README.md。

GitHub 存储库默认分支只包含网站的“源代码”，而不包含最终编译结果；每次推送更新时，自动化服务都会编译此源代码，并将编译结果存储在 gh-pages 分支中在 https://fortran-lang.org 提供。

因此，作为贡献者，您只需上传对站点源代码的更改而不是编译结果，因为这是从默认分支上的源代码自动构建的。

工作流程#

对站点的贡献是通过对 github 存储库的拉取请求进行的：https://github.com/fortran-lang/webpage/。

这样做的工作流程采用以下形式：

  1. 创建/更新网页的个人分支

    • （参见 github 帮助：同步 fork）

  2. 在你的 fork 中创建一个新分支

    • 分支名称应简明扼要地描述您的贡献，例如 fix-spelling-homepage, update-compiler-info

  3. 在本地分支上执行您的更改

  4. 将修改后的分支推送到 fork

    • 例如 git push --set-upstream origin fix-spelling-homepage

  5. 从修改后的 fork 分支在 fortran-lang/webpage 中创建拉取请求

    • （参见github帮助：创建拉取请求）

注意：在打开拉取请求之前，您必须使用 Sphinx 在本地构建您的更改（请参阅 README.md）以验证您的更改是否正确构建并按预期呈现。

注意：您可以在打开拉取请求后继续将更改推送到您的 fork 分支 —— 拉取请求将相应更新

您的拉取请求将由可能要求更改的社区其他成员审核。 GitHub 在其网站上提供了一个简单的界面，只需单击一个按钮即可应用（或拒绝）任何审阅者建议的更改。这避免了必须手动将建议复制到本地副本并再次推回。如果您使用“提交建议”按钮，如果您打算从计算机推送更多编辑，则需要使用 git pull 更新计算机上的本地副本。

一旦您的拉取请求获得批准，通常由至少两个其他社区成员批准，维护人员会将其合并到网页默认分支中，然后将其发布到 fortran-lang.org 站点。

如果需要，存储库维护人员可以构建您提议的更改的公共预览版，可在 fortran-lang.org/pr/#pr_id/ 查看，其中 #pr_id 是拉取请求的数字标识符。

这允许审阅者直接查看您的 PR 生成的结果。

**注意：**如果您将后续提交推送到您的拉取请求分支，您必须通过使用“#build_preview”评论拉取请求来重建拉取请求预览。

在合并并成功呈现拉取请求后，维护人员将删除预览版本。

**注意：**如果您的拉取请求预览链接不起作用或在重新构建后没有更新，请尝试在 URL 末尾添加一个随机参数，例如 https://fortran-lang.org /pr/98?v=2 —— 参数的名称和值无关紧要，但每次更新使用不同的值。这将迫使 GitHub 内容交付网络为您提供更新版本，而不是过时的缓存版本。

风格指南#

Markdown#

  • 将代码摘录放在代码块中，用反引号（```）。对内联代码摘录、编程语言关键字、变量名和文件名使用内联代码样式（`code`）。

  • 每个源代码行不超过一个句子，并且将长句子分成多行 —— 这对于避免 github 上的大 git diff 和代码审查块很重要。

外部链接#

建议在新选项卡中打开非现场超链接。在 Fortran-lang.org 上，所有此类链接都将自动以新标签图标为后缀；这让网站用户事先期望该链接将引导他们离开网站，同时在前一个选项卡中保持 fortran-lang.org 处于打开状态。

示例： 在新标签页中打开链接（HTML 或 markdown）

<a href="https://fortran-lang.discourse.group/" target="_blank" rel="noopener"
  >Discourse</a
>

内部网站链接#

指向 fortran-lang.org 网站其他部分的超链接应以 {{pathto('index',1)[:-5]}} 为前缀 - 这对于生成拉取请求预览很重要（请参阅 此处 进行解释）。

**示例：**markdown 链接

[Fortran-lang news]({{pathto('index',1)[:-5]}}News)

**示例：**html 链接

<a href="{{pathto('index',1)[:-5]}}Packages">Fortran packages</a>

页面内容#

有时为冗长的页面显示超链接的页面内容很有用。 inpage 目录树已自动生成，并将生成当前页面的目录。而在 fortran-lang.org 上生成整个路径目录的方法是：

**对于 RST 中的页面：**在目录索引页的末尾添加一个 toc 树指令，其中包含该目录中所有文件的名称。

.. toctree::
   :maxdepth: 2

   intro
   strings
   datatypes
   numeric

对于 MD 中的页面：

在 md 目录的索引页末尾添加 toc 树指令，其中包含该路径中所有文件的名称。

````{toctree}  
:hidden:
ARRAY_index 
````

代码风格#

使用两个空格缩进单元体，保持 contains 语句与其 module 或 type 处于同一级别。尝试将行长度限制为 90 个字符。这些考虑使得代码在具有较小宽度的设备上更具可读性且更容易查看。

module m
  implicit none
  private
  public :: a_t

  type :: a_t
    integer :: val
  contains
    procedure :: func
  end type a_t

contains

  subroutine func(self)
    class(a_t), intent(in) :: self
    if (self%val > 0) then
      print *, self%val
    end if
  end function func

end module m

每个代码块都应该有一个 0 级别的基本缩进，即使放在更大的上下文中它会缩进。

integer :: i1  ! yes
  integer :: i2  ! no

避免垂直对齐 :: 和行内注释，因为这会增加维护负担并在大多数情况下增加行长。

如果代码块包含无效的 Fortran 行，请将其保留为无语言代码块，以避免语法突出显示的红色框。

module <module name>
...
end module <module name>

在有助于提高可读性的表达式中随意省略空格，但通常在运算符周围应包含空格。

y1 = a * b
y2 = a*b + c*d  ! instead of a * b + c * d
y3 = a**2 + 1
y4 = (a*b + c*d) / 2
s3 = s1 // s2

通常在逗号后添加一个空格，除非使用短索引值或变量名称进行索引。

a(:,1)
a2(1:10, 2:5)
b(i,j)
b2(long_i_name, long_j_name)
b3(i + 2, j)
call some_subroutine(a, b, an_option=.false.)
c = [1, 2, 3, 10]
d = [(i, i = 1, 10)]
do i = 1, 10
! ...

除了简单索引之外，其他可以省略空格的情况：

  • import 中的别名

    use, intrinsic :: iso_c_binding, only: sp=>c_float, dp=>c_double

  • 字符串拼接

    print *, 'hello '//'world'

  • 访问派生类型的成员（属性）

    p%x
p%calc_something(a, b)

  • 传递关键字参数时使用 =

    call sr(a, b, c=3)
point = t_point(x=1., y=2.)
character(len=:), allocatable :: s

除了仅由一个词或一个短语组成的尾部行内注释，行内注释的第一个字母要大写。

! Compute new values
y = m*x + b  ! meters

这些代码格式类似于 DFTB+ 格式指南 中的建议。

文本#

对页面和章节标题使用句子大小写（而不是标题大小写）。

首次引入关键字/短语时，使用 emphasis（*emphasis*/_emphasis_，呈现为斜体）作为重点，… …

避免在段落中使用 strong（**strong**，呈现为粗体），因为粗体样式用于标题，请注意示例（Example:)、告诫/旁白等。

在适当的情况下使用警告/旁白（note、tip、important）。

  • 要将 ** 注释 ** 添加到 md 文档，请使用：

::::{note}
extra information, something that might appear in a footnote
:::::

  • 要将 ** 提示 ** 添加到 md 文档，请使用：

::::{tip}
information about best practices, practical tips
:::::

  • 要将 ** 重要 ** 文本添加到 md 文档中，请使用：

::::{importrant}
warnings, things to avoid, etc.
:::::

可以参考 Oxford comma，它通常使事情变得更加清晰。
